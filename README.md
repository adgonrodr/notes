# Create Your First Data Contract

Creating a data contract is essential for managing your data products, regardless of your tenant type. A data contract defines the structure, schema, and rules that govern data exchanges and management.

We provide translation scripts to help generate a template for your first data contract:

	•	Collibra Physical Metadata Translation: Tenants who have synced their physical metadata with Collibra can use a translation script to convert this metadata into a data contract template. This script extracts relevant information from Collibra and formats it into a data contract, simplifying the setup process.
	•	DM Configs Translation: Tenants using Tenant Engine v1.0 can leverage a translation script that converts existing DM configurations into a data contract. This helps ensure a smooth transition to the new framework without needing to redefine metadata.
	•	Manual Data Contract Creation: Tenants without synced physical metadata in Collibra or those not using Tenant Engine v1.0 will need to create a data contract manually. This involves defining the structure and schema directly within the data contract format, ensuring compliance with the necessary standards for testing, validation, and deployment.

For more details about the data contract structure and requirements, refer to the Data Contract Specification: Data Contract Specification Guide. This guide provides in-depth information to help you understand and create data contracts effectively.
